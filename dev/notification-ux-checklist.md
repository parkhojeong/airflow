# Notification UX Checklist

- [x] Provide an in-app notification UX across the Dashboard, Dag, and Asset pages.
- [x] Show pending HITL action notifications.
- [x] Show triggered deadline alert notifications.
- [x] Leave room for future important user notification types.
- [x] Open a modal when a user clicks a notification.
- [x] Provide a quick summary in the modal.
- [x] Provide an action in the modal.
- [x] Include a link to the related Dag or task for full context.
- [x] Provide a next action so users can quickly go through active notifications.
- [x] Let users review or handle active notifications without navigating through many separate pages.
